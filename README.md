# Прогнозирование спроса в розничной сети (Intermittent Demand Forecasting)

Прогнозирование дневных продаж (offline + online) на уровне **SKU × магазин × день** на горизонт **7 дней вперёд** с помощью CatBoost.

**WMAPE на тесте (сентябрь 2024): 50.5%** — прирост ~4.3 п.п. относительно baseline-лестницы (среднее 4 лагов = 54.8%).

---

## Данные

Датасет содержит данные о продажах розничной сети за 25 месяцев. Прогноз строится на сентябрь 2024 (тестовый период).

| Файл | Описание |
|------|----------|
| `sales.csv` | Офлайн-продажи (дата, товар, количество, цена, сумма, магазин) |
| `online.csv` | Онлайн-продажи |
| `markdowns.csv` | Продажи по markdown-ценам |
| `price_history.csv` | История изменения цен |
| `discounts_history.csv` | История промо-акций |
| `actual_matrix.csv` | Матрица ассортимента (какие товары были в продаже) |
| `catalog.csv` | Каталог товаров (департамент, класс, подкласс, тип, вес) |
| `stores.csv` | Справочник магазинов (город, формат, площадь, divisiоn) |

**Ключевая особенность:** intermittent demand — ~78% пар (SKU, магазин) имеют 0 продаж в конкретный день. Это определяет выбор метрик и архитектуры модели.

## Методология

### Пайплайн (8 шагов)

| Шаг | Что сделано |
|-----|-------------|
| **0. Рамка** | WMAPE как основная метрика, out-of-time сплит, multi-step (D+1…D+7) |
| **1. EDA** | Анализ распределений, сезонности, intermittent demand, корреляций |
| **2. Очистка** | Объединение online+offline, обработка пропусков и дубликатов |
| **3. Валидация** | Train: 10.01–17.08.2024, Val: 18–31.08, Test: 01–28.09.2024, 4 rolling-окна |
| **4. Baseline** | Лестница: mean → скользящее среднее → лаги → Prophet |
| **5. Признаки** | ~60+ фичей: лаги, rolling-статистики, календарные, ценовые, категориальные |
| **6. Отбор** | Permutation importance — удалены фичи-вредители |
| **7. CatBoost** | 7 моделей (по горизонтам), подобран learning_rate, depth, l2_leaf_reg |
| **8. Анализ** | Gain, SHAP, Permutation, PDP, ALE, калибровка, сегментные ошибки |

### Схема валидации

```
Train:                 |████████████████████████████|  Val  |    Test (4 окна)    |
                       2024-01-10              2024-08-17  08-18  09-01 ... 09-28
```

### Основные фичи

| Фича | Описание | Средний Gain |
|------|----------|:-----------:|
| `store_rolling_mean` | Средние продажи магазина за 28 дней | 17.8 |
| `rmean_28` | Скользящее среднее SKU за 28 дней | 15.1 |
| `lag_1` | Вчерашние продажи | 8.6 |
| `mom_ratio_7d` | Разгон/затухание: rolling-7d / rolling-7d (28д назад) | 6.8 |
| `item_store_hist_mean` | Историческое среднее по паре SKU×магазин | 6.3 |
| `hist_dow_mean` | Исторические продажи SKU в этот день недели | — |

## Результаты

### Итоговые метрики (тест, сентябрь 2024, Log-подход)

| Горизонт | WMAPE | Bias |
|:--------:|:-----:|:----:|
| H1 (D+1) | 0.489 | 0.79 |
| H2 (D+2) | 0.497 | 0.79 |
| H3 (D+3) | 0.509 | 0.77 |
| H4 (D+4) | 0.507 | 0.78 |
| H5 (D+5) | 0.509 | 0.78 |
| H6 (D+6) | 0.510 | 0.78 |
| H7 (D+7) | 0.518 | 0.78 |
| **Среднее** | **0.506** | **0.78** |

### Ключевые выводы

1. **`mom_ratio_7d` — хрестоматийный пример нелинейного признака.** Корреляция Пирсона r ≈ −0.009, но фича заняла #3 в permutation и SHAP — дерево ловит сигнал через пороги и взаимодействия.

2. **Bias ≈ 0.78 — систематический недопрогноз на ~22%.** Сентябрь по уровню продаж выше августа (сезонный рост), а бустинг не экстраполирует за пределы обучающей выборки.

3. **`hist_dow_mean` взрывается на H7.** На дальнем горизонте модель «выезжает» за счёт паттерна дня недели (Gain = 19.0), на H2–H5 эта фича — шум.

4. **`item_id` — скрытый гигант.** В Gain он лишь на 14-м месте, в SHAP — на 2-м. CatBoost строит вокруг `item_id` множество взаимодействий, распределённо формируя «профиль товара».

5. **Главная проблема — калибровка на хвосте.** Модель практически не выдаёт значения выше ~7.5 штук, хотя реальные продажи доходят до 57+. Log-трансформация не помогла, Tweedie loss провалился. Наиболее вероятное решение — two-stage модель (классификатор + регрессия).

## Как запустить

### 1. Установка зависимостей

```bash
python -m venv .venv
source .venv/bin/activate  # Linux/Mac
# .venv\Scripts\activate   # Windows
pip install -r requirements.txt
```

Основные зависимости: `pandas`, `numpy`, `catboost`, `scikit-learn`, `shap`, `matplotlib`, `seaborn`, `jupyter`.

### 2. Данные

Поместите CSV-файлы датасета в `data/`. Структура ожидается как в разделе «Данные».

### 3. Запуск ноутбука

```bash
jupyter notebook main.ipynb
```

Ячейки выполняются последовательно: загрузка данных → EDA → очистка → признаки → обучение → анализ.

### 4. Запуск per-model анализа

```bash
python model_analysis.py
```

Скрипт загружает обученные `.cbm` модели и генерирует все отчёты в `test/analysis/`.

## Идеи для улучшения

- **Two-stage модель:** бинарный классификатор (продажа > 0?) + регрессия на положительных — основной кандидат для решения проблемы калибровки
- **Sample weights** ∝ quantity — penalty за ошибки на крупных продажах
- **Quantile regression** — явное моделирование неопределённости
- **Horizon-aware фичи:** исключать `hist_dow_mean` из H2–H5, `rmean_14` из H1–H3
- **Экзогенные признаки:** промо-календарь, погода, праздники
